# Volcano 调度仿真系统 — 需求文档

**版本说明**：本文档描述当前仓库已实现能力，作为产品/交付侧需求与行为基线；实现细节以代码为准。

---

## 1. 目标与范围

### 1.1 建设目标

在本地以 **YAML 配置** 描述集群、Volcano 作业与调度插件，通过 **HTTP** 驱动内嵌 Volcano 框架的 **离散事件式调度仿真**，并输出 **任务清单、FlexNPU 利用率分析、结构化 CSV 报表**，用于调度策略验证与资源视图分析。

### 1.2 范围边界

| 包含 | 不包含 |
| --- | --- |
| Go 仿真进程（节点、Job、Task/Pod、调度 Session、Pod 阶段与仿真时钟） | 真实 K8s / Volcano 集群对接 |
| Python 客户端：配置转换、HTTP 调用、结果落盘与报表 | 修改 Volcano 调度算法内核（客户端侧不重写调度逻辑） |
| FlexNPU 相关注解解析、粒度取整、利用率/分配率估算与报表 | 生产级高可用、多租户隔离 |

---

## 2. 系统架构

- **Volcano_simulator**（Go，`cmd/sim`）：监听 HTTP（默认 **8006**），维护集群与作业状态，执行 `reset` / `step` / `stepResult`（及 `stepResultAnyway`）等接口。
- **Submit_volcano_workloads**（Python）：`SimRun.py` 为入口；`input_config/` 负责 YAML 加载与转换、`flexnpu_util_report` 与 `output_csv_reports` 负责统计与 CSV。

数据流：读取 `cluster` / `workload` / `plugins` YAML → `reset` 下发 → `step` 下发调度器配置并推进调度 → 轮询 `stepResult` 直至返回有效快照 → 写 `tasksSUM.csv`、`pod_phase_count.txt`、`flexnpu_utilization.txt` 及四类 CSV。

---

## 3. 配置输入需求

### 3.1 集群配置（`input_config/cluster/*.yaml`）

- 描述节点列表：名称、标签、注解、规格等。
- 支持 **FlexNPU** 相关节点注解，例如：
  - `volcano.sh/flexnpu-core.percentage-list`、`volcano.sh/flexnpu-memory.128mi-list`（按卡 ID 的容量映射 JSON）；
  - 拓扑等与仿真器约定一致的扩展字段（如 `topologies`）。

### 3.2 负载配置（`input_config/workload/*.yaml`）

- 顶层为 **JobList** 风格文档，内含多个 Volcano `batch.volcano.sh/v1alpha1` **Job**。
- **`spec.npuGranularityPercent`（可选）**  
  - 当 **大于 0** 时：仅对容器资源中的 **`volcano.sh/flexnpu-core.percentage`（flexnpu_core）** 的 **requests / limits** 做**向上取整**到粒度步长；**`volcano.sh/flexnpu-memory.128mi` 不参与取整**。  
  - 取整的同时：在 **`tasks[].template.metadata.annotations`** 写入  
    `volcano.sh/flexnpu-core.percentage-raw-by-container`（JSON：容器名 → 取整前数值），供报表区分「真实需求」与「取整后分配」。
- 任务模板支持 `tasks[].spec`；加载器会规范为 Volcano 兼容的 **`tasks[].template.spec`**（必要时自动补 `template` 外壳）。
- Job/PodGroup 侧可配置 **`volcano.sh/flexnpu-num`**（JSON：容器名 → 每容器占卡数），用于分卡估算。

### 3.3 插件与输出目录（`input_config/plugins/*.yaml`）

- **`scheduler`**：调度器配置（actions、tiers、插件参数等），作为 `/step` 请求体中的配置 YAML 字符串下发。
- **`output.outDir`**：结果根目录；支持 **`{date}`** 占位符，展开为 `年-月-日-时-分-秒` 时间戳；若为相对路径则相对客户端当前工作目录解析。

## 4. 仿真器（Go）功能需求

### 4.1 生命周期与 HTTP 接口

- **`/reset`**：接收节点 YAML、负载 YAML，初始化仿真环境；若仍有作业未完成可拒绝重置（返回约定错误码/标识）。
- **`/step`**：接收调度器配置，推进当前调度周期。
- **`/stepResult`**：在周期未结束且作业未完成时，可返回占位（如 `"0"`）表示快照不可用；否则返回包含 **Jobs、Nodes、仿真时钟** 等的 JSON（见 `simulator.Info`）。
- **`/stepResultAnyway`**：返回当前快照的变体（字段可能少于正式 `stepResult`）。

### 4.2 Job / Pod 建模

- 从 YAML 构建 Job 时，为每个 Task 生成 **Pod**；**合并 `Task.Template` 上的 `Annotations` 到 Pod**（再写入 Volcano 所需的 group 类注解），保证负载侧写入的 FlexNPU 原始 core 注解能进入 **stepResult** 中的 Pod。
- **Pod 创建时间（`CreationTimestamp`）**：在仿真将 Job **提交进集群**的时刻，对该 Job 下所有 Task **统一**写入当前仿真时钟（同一 Job 多副本时间戳一致）；**非**墙钟、**非**逐 Pod 独立生成。
- **Pod `status.startTime`**：在仿真逻辑中由 **Binding → Running** 等 transition 时设置，表示容器就绪/运行起始的仿真时刻。
- Pod 阶段与对外语义：未调度及 Binding 对外可为 **Pending**；Task 进入 Running 后对外为 **Running**（具体以 `main.go` 中 `syncSimulationPodPhases` 等为准）。

### 4.3 节点与调度

- 内嵌 Volcano 调度框架：Session、插件注册、`OpenSessionV2`、按配置的 action 链执行等。
- 节点可配置容器创建耗时、多容器间隔等仿真参数（与 `V2Node` 等结构一致）。

## 5. Python 客户端功能需求

### 5.1 入口（`SimRun.py`）

- 从配置文件路径加载 **cluster / workload / plugins**。
- 调用 **`reset`**、**`step`**，对 **`/stepResult`** 轮询直至获得字典形态快照。
- 将 workload 中的 **`npuGranularityPercent`** 注入快照（如 `npuGranularityPercent` 字段），保证与加载器取整规则一致的逐卡估算。
- 输出：**控制台摘要**、**tasksSUM.csv**、**pod_phase_count.txt**、**flexnpu_utilization.txt**、**write_output_config_csvs** 生成的 CSV。

### 5.2 配置加载（`input_config_loader`）

- `load_cluster_for_simulator` / `load_workload_for_simulator` / `load_plugins_for_simulator`：读 YAML 并转为仿真器约定的字符串或路径语义。
- `workload_input_to_simulator_yaml`：应用 **flexnpu_core 粒度取整**、**原始 core 注解**、`tasks[].spec` → **`template.spec`** 映射。
- `resolve_out_dir_pattern`：解析 `outDir` 中的 `{date}`。

### 5.3 FlexNPU 分析（`flexnpu_util_report`）

- 从 **stepResult** 解析节点注解中的按卡容量、Job 上 **flexnpu-num**、Running/Binding Pod 的容器 requests。
- **分卡策略**：按节点内卡 ID 列表，结合每容器占卡数 **轮询** 分配用量。
- **双轨计量**  
  - **利用率（raw）**：flexnpu_core 优先使用 Pod 注解 **`volcano.sh/flexnpu-core.percentage-raw-by-container`** 中的取整前数值；无注解时回退为 spec 中 request。  
  - **分配（granular）**：flexnpu_core 以 spec 中（已为加载器取整后的）request 为基础，再按快照中的 **`npuGranularityPercent`** 做与加载器一致的向上取整后分卡；**memory 不参与粒度**，分配与利用在数值上可一致。
- 提供 **`compute_flexnpu_snapshot`** 供 CSV 复用；**`format_flexnpu_report` / `print_flexnpu_utilization`** 生成文本报告。

### 5.4 CSV 报表（`output_csv_reports`）

在一次有效快照下，于结果目录 **平铺** 生成（无额外子目录）：

| 文件 | 内容要点 |
| --- | --- |
| **Node_desc.csv** | 节点级 flexnpu core/memory：**调度器 Used/Allocatable**（标量按内部约定缩放）的 allocated/total 与 **allocation_rate**；**逐卡估算汇总**的 utilized/total 与 **utilization_rate**。比率列为 **0–100 的数值，不带 `%` 后缀**；整数显示为整数（如 `100`）。 |
| **POD_desc.csv** | Running/Binding/Pending（已调度 Binding/Running 需有节点名）Pod：**flex 请求**、**phase**、**submit_time**（仿真提交时刻：metadata.creationTimestamp 等回退，**不含** startTime）、**status.startTime**（无则空）、**card_used_quantity**（JSON，分配侧按卡 core/mem）。 |
| **npu_chip.csv** | 按节点、按卡：**flexnpu_core_allocation_rate**、**flexnpu_core_utilization_rate**、**flexnpu_memory_allocation_rate**、**flexnpu_memory_utilization_rate**（0–100，两位小数，**无 `%`**）。 |
| **summary.csv** | 节点数、Running+Binding 计数、Pending 计数。 |

**前置条件**：快照中需含有效 **Nodes**；否则 `compute_flexnpu_snapshot` 失败，**不写**上述 CSV（`SimRun` 仍会写 tasks 与 flexnpu 文本，行为以代码为准）。

## 6. 非功能需求

- **运行环境**：Go 工具链可构建 `Volcano_simulator/cmd/sim`；Python 建议 3.8+，依赖含 `requests`、`PyYAML`、`prettytable` 等（以实际 import 为准）。
- **网络**：客户端与仿真器默认 **HTTP**、同机 **localhost:8006**（可在 `SimRun.py` 中修改）。
- **结果目录**：大体积结果目录宜在 **`.gitignore`** 中排除（如 `Submit_volcano_workloads/result/`）。

## 7. 术语与资源键名

| 术语 | 说明 |
| --- | --- |
| flexnpu_core | 资源名 `volcano.sh/flexnpu-core.percentage`（百分比类标量，具体含义与节点 cap 注解一致） |
| flexnpu_memory | 资源名 `volcano.sh/flexnpu-memory.128mi` |
| 提交时间 submit_time | 仿真将 Job 记入集群时写入的 Pod 创建时间语义，非 Running 起点 |
| status.startTime | Pod 进入 Running 相关路径时写入的仿真时刻 |
| 分配率 / 利用率 | core 在存在粒度时：分配侧对应取整后分卡量与容量比；利用侧对应「真实需求」分卡量与容量比（依赖原始 core 注解时） |
