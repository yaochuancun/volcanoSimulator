## 二、集群配置（cluster/*.yaml）

### 2.1 基础结构

```yaml
# 集群配置定义你的"虚拟数据中心"有哪些机器
nodes:
  - name: node-1                    # 机器名字（唯一）
    labels:                         # 标签（用于调度筛选）
      accelerator: npu
      rack: rack-a
    annotations:                    # 注解（存额外信息）
      # 这行最关键：定义有几张 GPU 卡，每张多少算力和显存
      volcano.sh/flexnpu-core.percentage-list: "[100, 100, 100, 100]"
      volcano.sh/flexnpu-memory.128mi-list: "[64, 64, 64, 64]"
    spec:
      unschedulable: false          # 是否可调度（维护模式设为 true）
    status:
      capacity:                     # 总容量
        cpu: "32"
        memory: "128Gi"
      allocatable:                  # 可用容量（减去系统预留）
        cpu: "28"
        memory: "120Gi"
```

### 2.2 FlexNPU 注解详解

**`volcano.sh/flexnpu-core.percentage-list`**
- 格式：JSON 数组
- 含义：每张 GPU 卡的算力百分比
- 示例：`"[100, 100, 100, 100]"` = 4 张卡，每张 100% 算力

**`volcano.sh/flexnpu-memory.128mi-list`**
- 格式：JSON 数组
- 含义：每张 GPU 卡的显存（单位：128Mi）
- 示例：`"[64, 64, 64, 64]"` = 每张卡 64×128Mi = 8GB 显存

**多节点示例：**
```yaml
nodes:
  # 高性能节点：8 张 A100，每张 100% 算力、80GB 显存
  - name: gpu-a100-x8
    labels:
      accelerator: npu
      npu-type: a100
    annotations:
      volcano.sh/flexnpu-core.percentage-list: "[100,100,100,100,100,100,100,100]"
      volcano.sh/flexnpu-memory.128mi-list: "[640,640,640,640,640,640,640,640]"
    status:
      capacity:
        cpu: "128"
        memory: "1024Gi"

  # 中性能节点：4 张 T4，每张 100% 算力、16GB 显存
  - name: gpu-t4-x4
    labels:
      accelerator: npu
      npu-type: t4
    annotations:
      volcano.sh/flexnpu-core.percentage-list: "[100,100,100,100]"
      volcano.sh/flexnpu-memory.128mi-list: "[128,128,128,128]"
    status:
      capacity:
        cpu: "32"
        memory: "128Gi"

  # CPU 节点：无 GPU
  - name: cpu-only
    labels:
      accelerator: none
    status:
      capacity:
        cpu: "64"
        memory: "256Gi"
```

### 2.3 节点标签的用途

**用于调度筛选：**
```yaml
# 集群配置
nodes:
  - name: node-1
    labels:
      accelerator: npu      # 有 GPU
      zone: zone-a          # 可用区

# 负载配置
jobs:
  - spec:
      affinity:
        nodeAffinity:
          requiredDuringSchedulingIgnoredDuringExecution:
            nodeSelectorTerms:
              - matchExpressions:
                  - key: accelerator
                    operator: In
                    values: ["npu"]
```

## 三、负载配置（workload/*.yaml）

### 3.1 基础结构

```yaml
# 负载配置定义你要跑什么任务
spec:
  npuGranularityPercent: 10       # 资源粒度（可选，默认 0）

jobs:
  - metadata:
      name: training-job-1
      namespace: default
      labels:
        sub-time: "0"             # 提交时间（秒），0 表示立即提交
        queue: default
    spec:
      minAvailable: 4             # Gang 调度：至少 4 个 Pod 才能启动
      schedulerName: volcano
      tasks:
        - name: train
          replicas: 4             # 启动 4 个 Pod
          template:
            metadata:
              annotations:
                volcano.sh/flexnpu-num: "2"  # 每个 Pod 用 2 张卡
            spec:
              containers:
                - name: train
                  image: pytorch/pytorch:latest
                  resources:
                    requests:
                      cpu: "4"
                      memory: "16Gi"
                      volcano.sh/flexnpu-core.percentage: "35"   # 每张卡 35% 算力
                      volcano.sh/flexnpu-memory.128mi: "64"      # 每张卡 8GB 显存
                    limits:
                      cpu: "8"
                      memory: "32Gi"
```

### 3.2 关键字段详解

**`spec.npuGranularityPercent`**
- 作用：设置 GPU 算力分配的最小粒度
- 示例：`10` 表示只能分配 10%、20%、30%... 不能分配 35%
- 如果任务请求 35%，会向上取整为 40%

**`metadata.labels.sub-time`**
- 作用：设置任务的提交时间（相对于仿真开始）
- 单位：秒
- 示例：`"0"` 表示第 0 秒提交，`"60"` 表示第 60 秒提交

**`spec.minAvailable`**
- 作用：Gang 调度参数
- 含义：至少要有这么多 Pod 才能启动
- 示例：`4` 表示 4 个 Pod 要么一起启动，要么都不启动

**`volcano.sh/flexnpu-num`**
- 作用：每个 Pod 用几张 GPU 卡
- 示例：`"2"` 表示这个 Pod 需要 2 张卡
- 总资源需求 = num × per-card-request

### 3.3 多任务场景

```yaml
spec:
  npuGranularityPercent: 10

jobs:
  # 任务 1：大模型训练（8 卡并行，立即提交）
  - metadata:
      name: llm-training
      labels:
        sub-time: "0"
    spec:
      minAvailable: 8
      tasks:
        - name: train
          replicas: 8
          template:
            metadata:
              annotations:
                volcano.sh/flexnpu-num: "1"
            spec:
              containers:
                - resources:
                    requests:
                      volcano.sh/flexnpu-core.percentage: "100"  # 独占整卡
                      volcano.sh/flexnpu-memory.128mi: "640"     # 80GB 显存

  # 任务 2：小模型训练（2 卡，30 秒后提交）
  - metadata:
      name: small-training
      labels:
        sub-time: "30"
    spec:
      minAvailable: 2
      tasks:
        - name: train
          replicas: 2
          template:
            metadata:
              annotations:
                volcano.sh/flexnpu-num: "1"
            spec:
              containers:
                - resources:
                    requests:
                      volcano.sh/flexnpu-core.percentage: "50"
                      volcano.sh/flexnpu-memory.128mi: "128"

  # 任务 3：推理服务（1 卡，60 秒后提交）
  - metadata:
      name: inference
      labels:
        sub-time: "60"
    spec:
      tasks:
        - name: serve
          replicas: 4
          template:
            metadata:
              annotations:
                volcano.sh/flexnpu-num: "1"
            spec:
              containers:
                - resources:
                    requests:
                      volcano.sh/flexnpu-core.percentage: "20"
                      volcano.sh/flexnpu-memory.128mi: "32"
```

### 3.4 不同优先级

```yaml
jobs:
  # 高优先级任务
  - metadata:
      name: urgent-job
    spec:
      priorityClassName: high-priority    # 引用 PriorityClass
      tasks:
        - replicas: 2
          template:
            spec:
              containers:
                - resources:
                    requests:
                      volcano.sh/flexnpu-core.percentage: "60"

  # 低优先级任务（可被抢占）
  - metadata:
      name: background-job
    spec:
      priorityClassName: low-priority
      tasks:
        - replicas: 4
          template:
            spec:
              containers:
                - resources:
                    requests:
                      volcano.sh/flexnpu-core.percentage: "30"
```

## 四、插件配置（plugins/*.yaml）

### 4.1 基础结构

```yaml
# 插件配置定义用什么调度策略
scheduler:
  actions: "enqueue, allocate, backfill"  # 调度动作顺序
  
  tiers:
    # 第一层：必需插件（硬性约束）
    - plugins:
        - name: gang          # Gang 调度
        - name: drf           # 主导资源公平
        - name: proportion    # 队列比例
    
    # 第二层：优化插件（软性约束）
    - plugins:
        - name: nodeorder     # 节点选择
        - name: binpack       # 紧凑打包

# 输出配置
output:
  outDir: "./result/{date}"   # 结果输出目录，{date} 会被替换为时间戳
```

### 4.2 Actions（调度动作）

**执行顺序很重要！**

```yaml
# 推荐配置 1：标准批处理
actions: "enqueue, allocate, backfill"
# enqueue:   将符合条件的 Job 加入队列
# allocate:  为任务分配节点
# backfill:  回填小任务到空闲资源

# 推荐配置 2：支持抢占
actions: "enqueue, allocate, preempt, backfill"
# preempt:   抢占低优先级任务的资源

# 推荐配置 3：预留资源
actions: "enqueue, reserve, allocate, backfill"
# reserve:   为重要任务预留资源
```

### 4.3 Plugins（调度插件）

**第一层：队列和资源管理**

```yaml
tiers:
  - plugins:
      # Gang 调度：确保一组任务同时启动
      - name: gang
        arguments:
          minJobExecutionTime: 10m    # 最小执行时间，防止频繁抢占
          
      # DRF：主导资源公平分配
      - name: drf
        arguments:
          preselectEnable: true       # 预筛选
          
      # Proportion：队列资源比例
      - name: proportion
        arguments:
          decayFactor: 0.9            # 衰减因子
```

**第二层：节点选择和优化**

```yaml
    - plugins:
        # NodeOrder：节点选择策略
        - name: nodeorder
          arguments:
            leastResourceWeight: 100   # 最少资源权重（spread）
            mostResourceWeight: 0      # 最多资源权重（binpack）
        
        # Binpack：紧凑打包（提高利用率）
        - name: binpack
          arguments:
            weight: 10                 # 权重
            
        # Predicates：硬性约束检查
        - name: predicates
          arguments:
            predicate.GPUSharingEnable: true   # 启用 GPU 共享
```

### 4.4 常见配置模式

**模式 1：最大化利用率（Binpack）**

```yaml
scheduler:
  actions: "enqueue, allocate, backfill"
  tiers:
    - plugins:
        - name: gang
        - name: drf
    - plugins:
        - name: binpack           # 紧凑打包
          arguments:
            weight: 100
```

**模式 2：负载均衡（Spread）**

```yaml
scheduler:
  actions: "enqueue, allocate, backfill"
  tiers:
    - plugins:
        - name: gang
        - name: drf
    - plugins:
        - name: nodeorder
          arguments:
            leastResourceWeight: 100   # 分散到不同节点
            mostResourceWeight: 0
```

**模式 3：支持抢占**

```yaml
scheduler:
  actions: "enqueue, allocate, preempt, backfill"
  tiers:
    - plugins:
        - name: gang
        - name: priority        # 优先级
        - name: drf
    - plugins:
        - name: preempt         # 抢占
        - name: nodeorder
```

**模式 4：队列隔离**

```yaml
scheduler:
  actions: "enqueue, allocate, backfill"
  tiers:
    - plugins:
        - name: gang
        - name: proportion      # 队列比例控制
          arguments:
            queue.capabilities: "[gpu]"
    - plugins:
        - name: nodeorder
```

## 六、配置验证清单

### 6.1 集群配置检查

- [ ] `nodes` 列表不为空
- [ ] 每个节点有 `name`
- [ ] `flexnpu-core.percentage-list` 格式正确（JSON 数组）
- [ ] `flexnpu-memory.128mi-list` 格式正确（JSON 数组）
- [ ] 两个列表长度一致（每张卡都有算力和显存）
- [ ] `capacity` 和 `allocatable` 合理

### 6.2 负载配置检查

- [ ] `jobs` 列表不为空
- [ ] 每个 Job 有 `metadata.name`
- [ ] `sub-time` 是数字字符串（如 `"0"`）
- [ ] `tasks[0].replicas` 是正整数
- [ ] `flexnpu-core.percentage` 在 0-100 之间
- [ ] `flexnpu-num` 是正整数
- [ ] 资源请求 ≤ 节点总容量

### 6.3 插件配置检查

- [ ] `actions` 不为空
- [ ] `tiers` 至少有一层
- [ ] 插件名称正确（gang、drf、binpack 等）
- [ ] `output.outDir` 路径合法

## 七、常见问题

### 7.1 "0" 响应

**问题：** reset 返回 "0"

**原因：** 上一次的仿真还没完成

**解决：**
```bash
# 方法 1：等待当前仿真完成
# 方法 2：重启 Go 仿真器
pkill sim
./sim
```

### 7.2 资源不够

**问题：** 任务一直处于 Pending

**排查：**
1. 检查节点总容量是否够
2. 检查 `minAvailable` 是否设置过大
3. 检查 Gang 调度条件是否满足

### 7.3 配置不生效

**问题：** 修改了配置但结果没变

**排查：**
1. 确认修改的是正确的文件
2. 确认 SimRun.py 中引用的是正确的路径
3. 重启 Go 仿真器确保状态清空

## 八、总结

**配置编写三步走：**

1. **集群配置**：定义你的"数据中心"
   - 多少台机器？
   - 每台多少 GPU？
   - 每张卡多少算力和显存？

2. **负载配置**：定义你的"任务"
   - 什么任务？
   - 什么时候提交？
   - 需要多少资源？

3. **插件配置**：定义"调度策略"
   - 用什么调度算法？
   - 优先利用率还是负载均衡？
   - 结果输出到哪里？

**配置即代码，版本化管理你的实验！**
