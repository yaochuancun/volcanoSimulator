## 一、快速启动（5分钟上手）

### 1.1 启动仿真器

```bash
# 进入仿真器目录
cd Volcano_simulator/cmd/sim

# 编译
go build -o sim .

# 运行（监听 localhost:8006）
./sim

# 看到 "simulator start..." 表示启动成功
```

### 1.2 运行实验

```bash
# 另开一个终端，进入客户端目录
cd Submit_volcano_workloads

# 安装依赖（首次）
pip install requests pyyaml prettytable

# 运行实验
python SimRun.py

# 结果会输出到 result/2026-xx-xx-xx-xx-xx/ 目录
```

### 1.3 查看结果

```bash
ls result/2026-04-07-12-00-00/
# tasksSUM.csv              - 任务清单
# pod_phase_count.txt       - 阶段统计
# flexnpu_utilization.txt   - GPU利用率分析
# Node_desc.csv            - 节点统计
# POD_desc.csv             - Pod详情
# npu_chip.csv             - 卡级统计
# summary.csv              - 汇总
```

## 二、配置文件模板

### 2.1 最小集群配置

```yaml
# cluster_minimal.yaml
nodes:
  - name: node-1
    labels:
      accelerator: npu
    annotations:
      volcano.sh/flexnpu-core.percentage-list: "[100, 100]"  # 2张卡，每张100%
      volcano.sh/flexnpu-memory.128mi-list: "[64, 64]"        # 每张8GB显存
    status:
      capacity:
        cpu: "16"
        memory: "64Gi"
```

### 2.2 最小负载配置

```yaml
# workload_minimal.yaml
spec:
  npuGranularityPercent: 10  # 10%粒度

jobs:
  - metadata:
      name: test-job
      labels:
        sub-time: "0"  # 立即提交
    spec:
      tasks:
        - name: train
          replicas: 2    # 2个Pod
          template:
            metadata:
              annotations:
                volcano.sh/flexnpu-num: "1"  # 每个Pod用1张卡
            spec:
              containers:
                - name: train
                  resources:
                    requests:
                      volcano.sh/flexnpu-core.percentage: "50"  # 每张卡50%
```

### 2.3 最小插件配置

```yaml
# plugins_minimal.yaml
scheduler:
  actions: "enqueue, allocate, backfill"
  tiers:
    - plugins:
        - name: gang
        - name: drf
    - plugins:
        - name: nodeorder

output:
  outDir: "./result/{date}"
```

## 三、常用配置模式

### 3.1 不同调度策略

**最大化利用率（紧凑打包）：**
```yaml
scheduler:
  actions: "enqueue, allocate, backfill"
  tiers:
    - plugins:
        - name: gang
        - name: drf
    - plugins:
        - name: binpack    # 紧凑打包
          arguments:
            weight: 100
```

**负载均衡（分散）：**
```yaml
scheduler:
  actions: "enqueue, allocate, backfill"
  tiers:
    - plugins:
        - name: gang
        - name: drf
    - plugins:
        - name: nodeorder
          arguments:
            leastResourceWeight: 100   # 分散
            mostResourceWeight: 0
```

**支持抢占：**
```yaml
scheduler:
  actions: "enqueue, allocate, preempt, backfill"  # 加了preempt
  tiers:
    - plugins:
        - name: gang
        - name: priority    # 优先级插件
        - name: drf
    - plugins:
        - name: preempt     # 抢占插件
        - name: nodeorder
```

### 3.2 不同资源粒度

| 粒度 | 配置 | 适用场景 |
|------|------|----------|
| 不取整 | `npuGranularityPercent: 0` | 理论研究 |
| 10% | `npuGranularityPercent: 10` | 推荐默认值 |
| 25% | `npuGranularityPercent: 25` | 粗粒度虚拟化 |

### 3.3 不同提交模式

**同时提交：**
```yaml
jobs:
  - metadata:
      labels:
        sub-time: "0"
  - metadata:
      labels:
        sub-time: "0"
```

**顺序提交：**
```yaml
jobs:
  - metadata:
      labels:
        sub-time: "0"     # 第0秒
  - metadata:
      labels:
        sub-time: "60"    # 第60秒
  - metadata:
      labels:
        sub-time: "120"   # 第120秒
```

## 五、常见问题速查

### 5.1 问题：reset返回"0"

**原因：** 上次的仿真还没完成

**解决：**
```bash
# 方法1：重启Go仿真器
pkill sim
./sim

# 方法2：等待当前仿真完成（如果快结束了）
```

### 5.2 问题：任务一直Pending

**排查步骤：**
1. 检查节点总容量是否够
2. 检查`minAvailable`是否设置过大
3. 检查Gang调度条件（需要同时满足）
4. 检查节点标签是否匹配

**调试：**
```python
# 在SimRun.py中打印详细信息
resultdata = client.get_json('/stepResult')
print("Nodes:", resultdata.get("Nodes"))
print("Jobs:", resultdata.get("Jobs"))
```

### 5.3 问题：配置不生效

**检查清单：**
- [ ] 修改的是正确的文件
- [ ] SimRun.py中引用的是正确的路径
- [ ] 重启了Go仿真器
- [ ] YAML格式正确（缩进、冒号后空格）

### 5.4 问题：Python报错

| 错误 | 原因 | 解决 |
|------|------|------|
| `ModuleNotFoundError` | 没装依赖 | `pip install requests pyyaml prettytable` |
| `Connection refused` | Go仿真器没启动 | 启动 `./sim` |
| `KeyError` | 配置格式错误 | 检查YAML字段 |

## 九、术语表

| 术语 | 含义 |
|------|------|
| **FlexNPU** | 可切分的GPU资源模型 |
| **Gang调度** | 一组任务要么全启动，要么全不启动 |
| **DRF** | 主导资源公平分配算法 |
| **粒度** | 资源分配的最小单位 |
| **利用率** | 原始需求 / 总容量 |
| **分配率** | 实际分配 / 总容量 |
| **Binding** | 已绑定节点，正在创建容器 |
| **Session** | 一次调度的上下文 |
