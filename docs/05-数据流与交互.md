# 05-数据流与交互：HTTP协议详解

> 本文档详细介绍 Python 客户端和 Go 仿真器之间的 HTTP 通信协议，包括接口定义、数据格式和交互流程。

---

## 一、通信架构

```
┌──────────────────────────────────────────────────────────────┐
│                    Python 客户端                             │
│                 (localhost: 任意端口)                          │
│                                                              │
│  ┌─────────────┐      ┌─────────────────────┐               │
│  │  JsonHttp   │──────│    requests.post    │               │
│  │  Client     │      │    requests.get     │               │
│  └─────────────┘      └─────────────────────┘               │
└──────────────────────────────────────────────────────────────┘
                           │
                           │ HTTP/1.1
                           │ Content-Type: application/json
                           ▼
┌──────────────────────────────────────────────────────────────┐
│                     Go 仿真器                                │
│                  (localhost:8006)                            │
│                                                              │
│  ┌──────────────────────────────────────────────────────┐   │
│  │              http.ListenAndServe(":8006")             │   │
│  │  ┌──────────┐  ┌──────────┐  ┌──────────────────┐   │   │
│  │  │ /reset   │  │ /step    │  │ /stepResult      │   │   │
│  │  │ Handler  │  │ Handler  │  │ Handler          │   │   │
│  │  └──────────┘  └──────────┘  └──────────────────┘   │   │
│  └──────────────────────────────────────────────────────┘   │
└──────────────────────────────────────────────────────────────┘
```

---

## 二、接口一览

| 接口 | 方法 | 功能 | 请求体 | 响应 |
|------|------|------|--------|------|
| `/reset` | POST | 初始化仿真 | JSON | JSON / "0" |
| `/step` | POST | 推进调度 | JSON | "1" |
| `/stepResult` | POST/GET | 获取结果 | JSON(可选) | JSON / "0" |
| `/stepResultAnyway` | GET | 强制获取结果 | - | JSON |

---

## 三、详细接口定义

### 3.1 POST /reset - 初始化仿真

**功能：** 重置仿真环境，加载新的集群和任务配置

**请求：**
```http
POST /reset HTTP/1.1
Host: localhost:8006
Content-Type: application/json

{
  "period": "-1",
  "nodes": "cluster:\n  - metadata:\n      name: node-1\n    ...",
  "workload": "jobs:\n  - metadata:\n      name: job-1\n    ..."
}
```

**字段说明：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `period` | string | 是 | 调度配置加载周期（秒），-1 表示只加载一次 |
| `nodes` | string | 是 | 集群配置的 YAML 字符串 |
| `workload` | string | 是 | 负载配置的 YAML 字符串 |

**响应（成功）：**
```http
HTTP/1.1 200 OK
Content-Type: application/json

{
  "Done": false,
  "V1Nodes": [
    {
      "metadata": {"name": "node-1"},
      "status": {"capacity": {...}, "allocatable": {...}}
    }
  ],
  "Clock": "0001-01-01 00:00:00 +0000 UTC"
}
```

**响应（失败 - 有任务在运行）：**
```http
HTTP/1.1 200 OK
Content-Type: text/plain

0
```

**注意事项：**
- 如果仿真器中有任务未完成，返回 "0" 表示拒绝重置
- 需要先等待当前仿真完成，或重启仿真器进程

---

### 3.2 POST /step - 推进调度

**功能：** 发送调度配置，触发一轮调度

**请求：**
```http
POST /step HTTP/1.1
Host: localhost:8006
Content-Type: application/json

{
  "conf": "actions: enqueue, allocate, backfill\ntiers:\n  - plugins:\n      - name: gang\n      ..."
}
```

**字段说明：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `conf` | string | 是 | 调度器配置的 YAML 字符串 |

**调度配置格式：**
```yaml
actions: "enqueue, allocate, backfill"
tiers:
  - plugins:
      - name: gang
      - name: drf
      - name: proportion
  - plugins:
      - name: nodeorder
      - name: binpack
```

**响应：**
```http
HTTP/1.1 200 OK
Content-Type: text/plain

1
```

**注意事项：**
- 返回 "1" 只表示配置已接收，不表示调度已完成
- 调度是异步执行的，需要轮询 `/stepResult` 获取结果

---

### 3.3 GET/POST /stepResult - 获取结果

**功能：** 获取当前仿真状态

**请求（GET）：**
```http
GET /stepResult HTTP/1.1
Host: localhost:8006
```

**请求（POST，带空 JSON）：**
```http
POST /stepResult HTTP/1.1
Host: localhost:8006
Content-Type: application/json

{
  "none": ""
}
```

**响应（调度完成）：**
```http
HTTP/1.1 200 OK
Content-Type: application/json

{
  "NotCompletion": false,
  "Nodes": {
    "node-1": {
      "Name": "node-1",
      "Allocatable": {...},
      "Used": {...},
      "Tasks": {...}
    }
  },
  "Jobs": {
    "default/job-1": {
      "Name": "job-1",
      "Tasks": {
        "task-0": {
          "Name": "task-0",
          "Status": "Running",
          "NodeName": "node-1",
          "Pod": {...}
        }
      }
    }
  },
  "Done": true,
  "V1Nodes": [...],
  "Pods": [...],
  "Clock": "0001-01-01 00:00:15 +0000 UTC"
}
```

**响应（调度中）：**
```http
HTTP/1.1 200 OK
Content-Type: text/plain

0
```

**关键字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `NotCompletion` | bool | 是否还有任务未完成 |
| `Nodes` | object | 节点状态，键是节点名 |
| `Jobs` | object | 任务状态，键是 "namespace/name" |
| `Clock` | string | 当前仿真时间 |
| `Done` | bool | 是否全部完成（NotCompletion 的取反） |
| `V1Nodes` | array | 精简的节点列表 |
| `Pods` | array | Pod 列表 |

---

### 3.4 GET /stepResultAnyway - 强制获取结果

**功能：** 无论调度是否完成，都返回当前状态

**请求：**
```http
GET /stepResultAnyway HTTP/1.1
Host: localhost:8006
```

**响应：**
```http
HTTP/1.1 200 OK
Content-Type: application/json

{
  "NotCompletion": true,
  "Nodes": {...},
  "Jobs": {...}
}
```

**与 `/stepResult` 的区别：**
- `/stepResult`：调度中返回 "0"，完成后返回 JSON
- `/stepResultAnyway`：始终返回 JSON（字段可能较少）

## 四、完整交互流程

### 4.1 时序图

```
Python 客户端                Go 仿真器
    │                            │
    │── POST /reset ────────────→│
    │   {nodes, workload}        │
    │                            │  初始化 ClusterInfo
    │                            │  加载 Nodes 和 Jobs
    │←─── JSON 确认 ─────────────│
    │                            │
    │── POST /step ─────────────→│
    │   {scheduler_conf}         │
    │                            │  解析调度配置
    │                            │  loadNewSchedulerConf = true
    │←─── "1" ──────────────────│
    │                            │
    │  (Go 主循环执行调度)         │
    │                            │
    │── GET /stepResult ────────→│
    │                            │  调度完成？
    │←─── "0" ──────────────────│  否，返回 "0"
    │                            │
    │  (等待 200ms)               │
    │                            │
    │── GET /stepResult ────────→│
    │                            │  调度完成？
    │←─── JSON 结果 ─────────────│  是，返回 JSON
    │                            │
    │  (Python 生成报表)           │
    │                            │
```

### 4.2 状态机

```
┌─────────────┐     reset      ┌─────────────┐
│   空闲       │───────────────→│  初始化中    │
│  (Idle)     │                │ (Initializing)
└─────────────┘                └──────┬──────┘
     ▲                                │
     │                                │ 完成
     │                                ▼
     │                          ┌─────────────┐
     │     ┌────────────────────│   就绪      │
     │     │   stepResult JSON  │  (Ready)    │
     │     │                    └──────┬──────┘
     │     │                           │ step
     │     │                           ▼
     │     │                    ┌─────────────┐
     │     │    stepResult "0"  │  调度中      │
     │     └────────────────────│ (Scheduling)│
     │                          └──────┬──────┘
     │                                 │
     └─────────────────────────────────┘
```

```json

      "Name": "node-1",
      "Allocatable": {
        "MilliCPU": 32000,
        "Memory": 137438953472,
        "ScalarResources": {
          "volcano.sh/flexnpu-core.percentage": 400000,
          "volcano.sh/flexnpu-memory.128mi": 256000
        }
      },
      "Used": {
        "MilliCPU": 8000,
        "Memory": 34359738368,
        "ScalarResources": {
          "volcano.sh/flexnpu-core.percentage": 100000,
          "volcano.sh/flexnpu-memory.128mi": 64000

      "Idle": {...},
      "Tasks": {
        "task-0": {...}
```

**注意：**
- 资源值是 **MilliValue** 格式（如 100% = 100000）
- Python 端需要除以 1000 转换为实际值

## 六、错误处理

### 6.1 常见错误

| 错误 | 原因 | 解决 |
|------|------|------|
| 连接 refused | 仿真器未启动 | 启动 Go 仿真器 |
| 返回 "0" | 有任务在运行 | 等待完成或重启仿真器 |
| JSON 解析失败 | 返回了 "0" | 检查 `str(result) == '0'` |
| 空 Nodes | 集群未正确加载 | 检查 cluster YAML |
| 空 Jobs | 负载未正确加载 | 检查 workload YAML |

### 6.2 重试策略

```python
class JsonHttpClient:
    def get_json(self, path, json=None, max_retries=10):
        for attempt in range(max_retries):
            try:
                response = requests.post(url, json=json, timeout=10)
                response.raise_for_status()
                return response.json()
            except requests.RequestException:
                if attempt == max_retries - 1:
                    raise
                time.sleep(0.1)  # 退避等待
```

## 八、总结

**HTTP 接口设计原则：**
1. **简单**：4 个接口覆盖全部功能
2. **异步**：step 和 stepResult 分离，支持长耗时调度
3. **容错**：失败返回 "0" 而不是报错
4. **完整**：返回完整的集群状态，便于分析

**关键要点：**
- `/reset` 初始化，返回 JSON 或 "0"
- `/step` 触发调度，返回 "1"
- `/stepResult` 轮询结果，返回 JSON 或 "0"
- 资源值是 MilliValue，需要除以 1000
